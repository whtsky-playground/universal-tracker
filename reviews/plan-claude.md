# 设计方案：环球影城年卡回本追踪器 v4

> 审稿人：Claude（创意总监视角）
> 基于：`src/index.html` — 收藏级票根版本

---

## 一、现版本分析

### 做对了什么（必须保留的核心资产）

1. **票根隐喻贯穿始终**——主题乐园追踪器用票根做视觉母题，天然契合。圆形缺口、虚线穿孔、条形码、印章，每个细节都在强化同一个故事。
2. **巨型百分比是社交货币**——96px 的数字 + 斜角印章，一截屏就能传达"我赚了/快赚了"，是天然的朋友圈素材。这个 hero 区是整个产品的灵魂。
3. **信息层级做得克制**——最重要的（回本百分比）最大，次要的（净值/次数/天数）一行带过，明细藏在折叠里。不贪心。
4. **纸质温度感**——暖色纸纹 + Caveat 手写体 + 微旋转，让数据仪表盘有了"手账"的亲切感，和环球影城的快乐调性一致。
5. **渐进展开**——入园战报默认只显示 4 条、消费明细默认折叠 2 组，不会一上来就信息过载。

### 问题在哪里（改进空间）

| # | 问题 | 严重度 | 说明 |
|---|------|--------|------|
| P1 | **页面是一根无限长的面条** | 高 | 7 个区块纵向堆叠，没有节奏感。滑到底部时已经忘了开头。缺少"呼吸感"——每个 section 的视觉权重几乎一样。 |
| P2 | **超过 100% 之后体验断崖** | 高 | 回本前体验紧张刺激，回本后百分比继续涨到 200%、300%，但没有新的叙事。印章从 BREAK EVEN 变成 PROFIT MODE 就结束了。"赚后"阶段缺乏成就系统。 |
| P3 | **资金流向图谱可读性差** | 中 | 双向条形图的中轴线概念好，但"入园"只有右侧有条、其他只有左侧有条，看起来像错位了。且分类名只有 42px 宽，挤。 |
| P4 | **Canvas 趋势图与纸质美学断裂** | 中 | 图表用 Canvas 绘制，视觉上突然变成了"数据产品"风格，和周围的手账/票根气质不搭。斜线填充纹理是个好尝试，但不够。 |
| P5 | **三个 mini-ticket 太通用** | 低 | Saved / Spent / Avg 三张小卡视觉上像任何金融 App 都会有的 KPI 卡，没有票根的灵魂。微旋转是唯一的特色。 |
| P6 | **入园战报卡片重复感强** | 中 | 展开后 10+ 张一模一样的卡片，只有数字不同。缺少区分度和叙事线——没有"故事弧"。 |
| P7 | **截图体验只覆盖 hero** | 中 | "长按保存"暗示了截图分享需求，但只有 hero 区适合截图。如果我想分享完整战绩？或者单次最佳战报？没有方案。 |
| P8 | **"赛季"概念有框架但没落地** | 低 | Season 1 badge 和 season range 只是装饰，没有赛季对比、赛季总结、赛季成就。 |

## 二、设计方向：「年卡护照」

### 核心概念

从"一张票根"进化为"一本护照"。

物理隐喻升级：票根是一次性的——看完就扔；**护照是积累型的**——每一页都是一段旅程，盖满了章才有成就感。这个隐喻天然适合年卡这种"长周期、多次使用、逐渐积累"的产品。

但我们不是要推翻票根，而是**让票根成为护照的封面**。

### 信息架构（从上到下）

```
┌──────────────────────────────────┐
│  A. 护照封面（= 现有 Hero 票根）    │  ← 截图区，核心社交货币
│     保留巨型百分比 + 印章           │
│     增加：动态封面质感               │
├──────────────────────────────────┤
│  B. 快速概览条                      │  ← 替代 mini-tickets
│     三个数字一行排列，更紧凑          │
├──────────────────────────────────┤
│  C. 成就墙（= 进化版赛季进度）       │  ← 用印章集替代进度条
│     护照签证页 → 盖章               │
├──────────────────────────────────┤
│  D. 旅途地图（= 进化版趋势图）       │  ← 用 SVG 替代 Canvas
│     路线图风格，站点标记             │
├──────────────────────────────────┤
│  E. 每次入园 = 一页护照              │  ← 替代入园战报卡片
│     翻页/手风琴交互，每页有独立个性   │
├──────────────────────────────────┤
│  F. 消费指纹（= 进化版资金流向）     │  ← 环形图替代双向条形图
│     视觉更直觉，占地更小             │
├──────────────────────────────────┤
│  G. 护照末页（尾声）                │  ← 替代 season footer
│     下一目标 + 签名                  │
└──────────────────────────────────┘
```

## 三、逐区块详细设计

### A. 护照封面（Hero）

**保留什么：** 巨型百分比数字、印章叠加、虚线穿孔、条形码、品牌信息。这些全部保留，它们是产品的灵魂。

**改进点：**

1. **百分比显示策略改变**
   - 回本前：显示百分比（如 `73.2%`），和现在一样
   - 回本后：切换为显示净赚金额（如 `+¥582`），用金色渐变，配合「PROFIT」印章。不再显示 130%、200% 这种失去直觉意义的数字
   - 这解决了 P2：回本后有明确的新叙事——"我赚了多少"

2. **封面材质升级**
   - 现有纸纹保留但降低一档
   - Hero 区增加**烫金边框线**——用 `linear-gradient` 模拟的细金线沿 ticket 边缘走一圈，1px，若隐若现
   - 印章增加**蜡封质感**——stamp 背景加径向渐变 + 内阴影，模拟蜡封的凸起感

3. **动态印章系统扩展**
   现有四档（WARM-UP / ON FIRE / BREAK EVEN / PROFIT MODE）改为：

   | 阶段 | 印章文字 | 印章颜色 | 视觉处理 |
   |------|---------|---------|---------|
   | 0-25% | `ROOKIE` | 灰蓝 | 浅印，低透明度，像刚盖上去墨还没干 |
   | 25-50% | `REGULAR` | 蓝色 | 正常墨迹 |
   | 50-75% | `VETERAN` | 紫色 | 加粗边框 |
   | 75-99% | `ALMOST THERE` | 橙金 | 边框开始发光（box-shadow pulse） |
   | 100-149% | `BREAK EVEN` | 绿色 | 双层印章（内外框） |
   | 150%+ | `LEGEND` | 金色 | 烫金效果 + 微光动画 |

4. **条形码变为真实可扫描的 Code128**
   - 编码内容：`USJ-2025-007`（护照编号）
   - 扫出来跳转到这个页面本身——用户分享截图，别人扫码能看到（如有公开部署）
   - 即使不做真实跳转，真实条形码比装饰性条纹更有"实物感"

### B. 快速概览条

**替代现有 mini-tickets**，解决 P5。

**设计：** 一行三列，不再用卡片，改用**护照信息页**风格——像护照个人信息页上的字段布局：

```
┌────────────────────────────────┐
│  SAVED        SPENT       NET  │
│  ¥1,820       ¥380      ¥1,440│
│  · · · · · · · · · · · · · ·  │  ← 微型穿孔分隔线
│  8 visits  ·  62 days  ·  ¥180/visit │
└────────────────────────────────┘
```

- 第一行：大字段（三个核心金额）
- 分隔线：点状虚线（模拟护照页的安全线）
- 第二行：辅助信息（次数、天数、单次均值）
- 整体是一张扁平卡片，不做旋转，信息密度更高
- 字体全部用 `IBM Plex Mono`，数字用 `Bebas Neue`

### C. 成就墙（护照签证页）

**替代现有的赛季进度条**，同时解决 P2 和 P8。

**核心理念：** 进度条是线性的、无趣的。护照签证页上的印章是**面积型的、可收集的**——视觉冲击力完全不同。

**设计：**

```
┌────────────────────────────────┐
│  STAMPS COLLECTED   [4/10]     │
│                                │
│  🔥         ⚡        🎯       │
│  首次入园    五次常客   十次元老  │
│  2025.01    2025.03   ----     │
│                                │
│  🏆         💰        🌙       │
│  回本成功    赚回 ¥500  夜场达人  │
│  2025.04    ----      2025.02  │
│                                │
│  🎪         🍔        🎁       │
│  活动全勤    美食猎人   购物狂    │
│  ----       2025.03   ----     │
│                                │
└────────────────────────────────┘
```

- **3x3 网格**（或 3x4），每格是一个成就印章位
- 已达成：彩色印章，有日期，有轻微旋转（-2~2deg 随机），有印泥质感
- 未达成：灰色轮廓，虚线边框，标题可见但日期显示 `----`
- 每个印章是一个 60x60 的圆角矩形，内含 emoji + 标题 + 日期
- 解锁时动画：印章从上方"盖下来"（translateY + scale + opacity），配合一个微妙的震动（CSS shake）

**成就列表设计（10 个基础成就）：**

| 成就 | 条件 | 图标 |
|------|------|------|
| 首次入园 | 第 1 次入园 | 🎫 |
| 五次常客 | 累计 5 次 | ⭐ |
| 十次元老 | 累计 10 次 | 👑 |
| 回本成功 | 净值 >= 年卡价格 | 🏆 |
| 赚回五百 | 净值超出 ¥500 | 💰 |
| 单日之王 | 单日净值 > ¥300 | 🔥 |
| 美食猎人 | 餐饮消费 >= 3 次 | 🍔 |
| 购物达人 | 商品消费 >= 3 次 | 🎁 |
| 周末战士 | 周末入园 >= 5 次 | ⚡ |
| 全勤选手 | 连续 4 周都去了 | 🎪 |

这个系统让回本后依然有追踪动力（解决 P2），同时比进度条更有"收集感"。

**进度条不删除**，缩小为成就墙下方的辅助信息——一行文字 + 极细进度条即可。

### D. 旅途地图（趋势图重构）

**替代现有 Canvas 趋势图**，解决 P4。

**核心变化：Canvas → SVG + CSS 动画**

理由：
- SVG 天然支持 DOM 事件、CSS 样式，和页面其他元素的视觉语言可以统一
- 不需要处理 DPR 缩放
- 更容易实现"纸质手绘"风格

**视觉风格：「手绘路线图」**

```
       ¥1688 ·····🏁 GOAL ·····
              ╱
            ╱   ← 虚线投影
          •⑦
        ╱
      •⑥
    ╱    ╲
  •④    •⑤
  ╱
•③
╱
•②
╱
•① START
```

- 线条用 `stroke-dasharray` 动画，模拟手绘笔迹逐渐画出
- 每个数据点是一个"驿站"圆点，hover 弹出 tooltip
- 目标线用**红色虚线 + 小旗帜 emoji**标注
- 回本交叉点用**金色星标**突出
- 区域填充改用 CSS 渐变（不再用 canvas pattern），配合 SVG clipPath
- Best/Worst 点用绿/红色加重，旁边有小标签

**动画策略：**
- 页面进入视口时，SVG path 的 `stroke-dashoffset` 从全长过渡到 0，线条"画"出来，耗时 2s
- 每个驿站点在线条画到它时才 `scale(0) → scale(1)` 弹出
- 使用 IntersectionObserver 触发——不在视口内不动画

### E. 每次入园 = 一页护照（入园记录重构）

**替代现有入园战报卡片列表**，解决 P6。

**核心变化：从"列表"变为"翻页"**

物理隐喻：护照的每一页是一次出入境记录。

**方案一（推荐）：堆叠卡片 + 点击展开**

默认只显示最近一次入园的完整卡片，其他以"叠在下面"的方式露出顶部边缘（类似 iOS 钱包里的叠卡）。点击某一张，它滑上来展开。

```
┌──────────────────────────────┐
│ #7  3月1日 周六               │  ← 完整展开的最近一次
│ +¥320                        │
│ ██████████████░░░░            │
│ 入园 +280 · 餐饮 -40 · ...    │
├──────┤                        │
│ #6  ──┤  ← 叠在后面，只露标题行
├──────┤
│ #5  ──┤
├──────┤
│ #4  ──┤
│ ... 点击展开全部               │
└──────────────────────────────┘
```

- 每张卡片增加**个性化元素**：
  - 根据当日净值显示不同的心情 emoji（大赚 🤩 / 小赚 😊 / 小亏 😅 / 大亏 💸）
  - 根据星期显示不同背景色调（周末偏暖、工作日偏冷）
  - BEST 和 WORST 保留特殊标记

- **单次卡片内部布局**重新设计：
  ```
  ┌─────────────────────────────┐
  │  #7          2025.03.01 SAT │  ← 编号 + 日期 + 星期
  │─────────────────────────────│
  │        +¥320  🤩            │  ← 净值 + 心情
  │  ████████████░░░░░          │  ← 收支堆叠条
  │  +280 入园  -40 餐饮         │  ← 明细摘要（最多两行）
  │─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ │
  │  累计: ¥1,280 / ¥1,688      │  ← 当日累计进度
  └─────────────────────────────┘
  ```

### F. 消费指纹（资金流向重构）

**替代现有双向条形图**，解决 P3。

**方案：并排双半环**

```
        SAVED              SPENT
     ┌────────┐        ┌────────┐
     │  🎢    │        │  🍔    │
     │ ¥1,820 │        │  ¥380  │
     │  ◠◠◠   │        │  ◠◠◠   │
     └────────┘        └────────┘
```

更具体地说：

- 左侧：**收入半环图**（只有入园一个来源，所以是满环），中间显示总额
- 右侧：**支出半环图**（多个分类用不同颜色弧段），中间显示总额
- 环形图下方：分类图例，每个分类一行——`emoji 名称 ¥金额 占比%`
- 用 SVG `<circle>` + `stroke-dasharray` 实现，比 canvas 更轻量

**备选方案（如果觉得双环太花哨）：**

简化为**两列带图标的条形图**——左列"收入"、右列"支出"，每条前面是 emoji 图标，后面是金额。不用双向轴，直接分开，更好理解。

## 四、动效策略总纲

### 原则

1. **有因才有果**——每个动画必须有信息传达目的，不为炫技
2. **入口动画一次性**——首次加载时播放，后续交互不重复
3. **滚动触发，不预加载**——用 IntersectionObserver，进入视口才触发
4. **150ms-2000ms 区间**——微交互 150-300ms，叙事动画 1-2s，不超过 2s

### 具体动画清单

| 区块 | 动画 | 时长 | 触发条件 |
|------|------|------|---------|
| Hero 百分比 | 数字 countUp | 1.4s | 页面加载 |
| Hero 印章 | scale(0.6)→scale(1) + rotate 微调 | 0.4s | 百分比 countUp 完成后 |
| 概览条数字 | countUp | 1s | 页面加载（与 Hero 同时） |
| 成就印章 | translateY(-20px)→0 + scale(1.1)→1 + opacity | 0.3s/个 | 滚动进入视口，逐个延迟 0.1s |
| 趋势图路线 | stroke-dashoffset 擦除 | 2s | 滚动进入视口 |
| 趋势图驿站点 | scale(0)→scale(1) | 0.2s/个 | 路线画到该点时 |
| 入园卡片展开 | height auto + translateY | 0.3s | 用户点击 |
| 消费环形图弧段 | stroke-dashoffset | 1.2s | 滚动进入视口 |
| 回本庆祝 | 升级版 confetti（环球影城主题色 🟡🔴🟢🔵） | 3s | 页面加载 + 已回本 |

### 减弱动效

保留 `prefers-reduced-motion` 支持。检测到时：
- 所有动画 duration 降到 0.01ms
- countUp 直接显示最终值
- confetti 不播放

## 五、视觉语言升级

### 色彩

保留现有色彩系统（暖纸色 + 绿/红/金/蓝），但增加：
- **护照蓝** `#1a365d` — 用于护照边框、MRZ 区域文字，增加"证件感"
- **蜡封红** `#8b2020` — 用于成就印章的特殊变体，比现有 stamp-red 更深沉
- **签注紫** `#6b21a8` — 用于 VETERAN 阶段印章

### 字体

在现有基础上：
- `Bebas Neue` — 继续做大数字
- `IBM Plex Mono` — 继续做数据标签
- `Caveat` — 继续做手写感文字
- 新增 `OCR-B`（或退而求其次用等宽字体模拟）— 用于 MRZ 机读区

### 质感

- **纸纹**：保留但降低 10% 不透明度，避免在印章/图表区域干扰可读性
- **烫金**：用 `linear-gradient(135deg, #c87f0a, #f0d060, #c87f0a)` + `background-clip: text` 实现
- **蜡封**：用 `radial-gradient` + `box-shadow: inset` 模拟
- **印泥**：现有 SVG noise texture 保留，是好细节

### 边框与分隔

## 七、技术约束与建议

1. **继续单文件 HTML**——这是项目的核心约束和优势（Notion 数据注入 `__DATA__`），不要引入构建工具
2. **Canvas → SVG**——趋势图改用 SVG，减少代码量，提升可维护性
3. **IntersectionObserver**——替代当前的 setTimeout 触发策略，更优雅
4. **CSS 变量继续使用**——现有变量系统很好，扩展而不是重写
5. **新增字体控制在 1 个以内**——避免加载过多 Google Fonts，OCR-B 用系统等宽体回退即可

## 九、最后的话

现有版本已经是一个 **90 分的作品**。票根隐喻、巨型百分比、温暖纸质感——这些是竞品不可能有的差异化。

这个方案不是推翻，是**把一张好票根装订成一本好护照**。核心赌注押在两点：

1. **成就墙**给了用户回本之后继续打开的理由
2. **护照隐喻**让每次入园都有"盖章"的仪式感

好的设计不是功能越多越好，而是让用户每次打开都有一点点期待。
