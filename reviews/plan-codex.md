# 收藏级票根 2.0 设计方案（Codex）

## 1. 现有版本分析（src/index.html）

### 1.1 主要优点

1. 叙事感强，主题统一
- 用“票根”语义把数据产品做成了可收藏内容：齿孔、印章、条码、手写 slogan、纸张噪点都在服务同一个世界观。
- 情绪曲线清晰：从 Hero 百分比到赛季进度、再到战报与账本，形成“成绩揭晓 -> 过程复盘”的阅读路径。

2. 信息组织成熟
- 顶部 KPI（回本百分比/净值/次数/持卡天数）一屏完成核心判断。
- 中段“轨迹图 + 战报 + 资金流向”提供解释层，底部账本给到审计层，完整闭环。

3. 动效有“仪式感”
- 计数器、里程碑弹跳、流向条展开、回本撒花都能强化“成就达成”的反馈。
- `prefers-reduced-motion` 已兼顾可访问性，工程意识较好。

4. 细节完成度高
- 文案语气统一（战报/赛季/进度），并且根据进度动态变化。
- 移动端参数、字体组合、触摸 hover 兼容都考虑得比较细。

### 1.2 主要问题与机会点

1. 视觉密度偏高，层级有竞争
- 纸张纹理、印章、卡片倾斜、图表标注、贴纸等元素都很抢眼，局部存在“每个都想当主角”的情况。
- `mini-tickets + season-card + trend + visit + flow + ledger` 在小屏连续堆叠后，阅读疲劳明显。

2. 色彩语义有轻微冲突
- 页面以暖纸色为基底，但类别色（紫、蓝、粉、青等）较分散，导致“收藏品高级感”会被“信息看板感”稀释。
- 绿色既代表“收益”又承担大量主视觉元素，长期看会降低关键状态的辨识度。

3. 图表可读性可再提高
- 趋势图注释多、点位标签多，和移动端手势提示叠加后，认知负担较大。
- 多次入园后，横轴标签/tooltip 可用性下降。

4. 情绪高潮集中在“已回本”用户
- 回本前后的体验差异偏大：完成态有撒花和印章奖励，未完成态激励机制略弱。
- 对“长期收藏”而言，目前更像一张“实时战绩票”，还不是“年度纪念票”。

---

## 2. 新方向提案：**《档案馆票证 / Archive Pass》**

目标：保留“收藏级票根”基因，但从“战绩海报”升级为“可长期保存的个人赛季档案”。
关键词：**档案感、克制高级、时间沉淀、可打印收藏**。

### 2.1 视觉语言（全局）

1. 设计母题
- 从“游乐场当日票根”转向“博物馆档案卡 + 旅行证章”。
- 画面气质从热闹庆典改为“克制但有纪念价值”。

2. 色彩系统（收敛到 1 主 2 辅 + 功能色）
- 主色：`Archive Navy`（深墨蓝，承担标题/关键线条）
- 辅色：`Aged Ivory`（纸底）、`Brass Gold`（里程碑/奖章）
- 功能色：收益绿、支出红仅用于数值，不再大面积铺色
- 类别色由“高饱和块”改为“同色相明度层级 + 图标区分”

3. 字体策略
- 标题数字：保留高识别数字字体（延续收藏票根识别）
- 正文：人文黑体（稳定可读）
- 装饰字：仅保留 1 处手写体（签名或寄语），避免多处手写造成风格噪声

4. 质感策略
- 保留纸纹，但降噪（不超过当前体感的 60%）。
- 加入“压印/凹凸边框”替代部分高对比描边，让高级感来自材质，不来自堆叠装饰。

## 3. 页面结构重构（每个区块如何设计）

### 3.1 区块 A：封面票证（Hero）

1. 信息层级
- L1：`回本进度 %`
- L2：`累计净值` + `距目标差额/超额`
- L3：`票号 / 更新时间 / 卡种`

2. 布局
- 采用“左右双栏票证”：左侧大数字（进度），右侧为印章区与元信息。
- 原条码改为“档案编号条 + 微型防伪纹样”，更收藏化，弱化“截图工具感”。

3. 视觉细节
- 印章由“彩色盖章”改为“铜箔压章”风格，分三态：`IN PROGRESS / BREAK EVEN / PROFIT`。
- 去掉过多倾斜角度，仅保留轻微 0.5-1deg 不规则，模拟真实纸张摆放。

4. 动效
- 首屏只保留 2 个动作：百分比滚动 + 印章轻压入场（120-180ms）。
- 限制首屏并行动画数量，确保“先读数据，再看装饰”。

### 3.2 区块 B：赛季时间轴（替代当前 season-card + mini-tickets）

1. 设计思路
- 合并现有“赛季进度条”和“3 张 mini-ticket”，改为一条纵向时间轴模块，减少卡片碎片化。

2. 内容结构
- 时间轴节点：`25/50/75/100%`
- 每节点右侧显示“达成日期（已达成）或预计日期（未达成）”
- 轴底部放 3 个关键指标 chip：`总节省 / 总消费 / 单次净值`

3. 视觉语言
- 节点图形从 emoji 改为统一徽章（线框圆章），到达后填充金属色。
- 线条用“档案蓝细线 + 金色高亮段”，避免当前高饱和渐变的运动感过强。

4. 动效
- 进入视口后时间轴按节点顺序点亮；每个节点延迟 80ms。
- 未达成节点只做 opacity 过渡，不做弹跳。

### 3.3 区块 C：回本曲线（趋势图升级）

1. 图表目标
- 从“信息多而炫”改为“单图讲清一个结论”：你离回本还有多远，以及变化是否稳定。

2. 图层精简
- 保留：累计净值主线、目标线、回本点标记。
- 删除：BEST/WORST 文本浮标常显，改为点击点位后在下方信息条显示。

3. 交互
- 移动端采用“吸附点位”的拖拽十字线；tooltip 固定在图表下方，避免挡线。
- 提供双模式切换：`累计净值` / `单次净值`（分段柱），便于看波动。

4. 视觉
- 线条使用单色深浅层级；仅在回本点使用金色。
- 网格线更浅，y 轴标签减少到 3 档，提高清洁度。

### 3.4 区块 D：单次入园战报（卡片列表）

1. 信息层级
- 每卡只展示：日期、净值、节省/支出占比条。
- BEST/WORST 保留但改为“侧边签条”，避免大贴纸打断排版。

2. 交互
- 默认展示最近 4 次；展开后按月份折叠分组（例如 `2026.02`）。
- 点击卡片进入“当日明细抽屉”，而不是把所有明细直接铺在页面里。

3. 视觉
- 卡片背景统一为浅纸底，特殊卡只改边框和签条颜色，不改整张底色。
- 数值统一右对齐，让纵向扫读更快。

4. 动效
- 卡片出现采用轻微 y 轴上浮 + 淡入；展开时使用高度过渡。
- 不做 hover 旋转，移动端意义不大且会增加漂浮感。

### 3.5 区块 E：资金流向（从“对抗条”改为“账户结构图”）

1. 当前问题
- 双向条表达新颖但认知成本偏高，尤其初看难懂“左支右收”的语义。

2. 新方案
- 改成“甜甜圈 + 排名表”双层：
  - 上层甜甜圈显示支出构成占比（餐饮/商品/停车...）
  - 下层单独一行展示“入园节省总额”作为对照主指标
- 或在窄屏下仅保留“排名表 + 进度条”，可读性优先。

3. 视觉
- 分类颜色降饱和并统一色阶，同一类别在全站一色到底。
- Icon 仅作辅助，不再成为主辨识手段。

### 3.6 区块 F：完整账本（明细）

1. 结构
- 改成“档案折页”样式：默认只显示最近 2 次 + 搜索/筛选入口（日期、类别）。
- 展开后支持快速定位“回本当天记录”。

2. 关键信息增强
- 每天头部增加“当日 ROI（净值/入园票价）”小标签。
- 明细行加入“原价 -> 实付/节省”对照格式，增强说服力。

3. 视觉
- 减少分割虚线密度，改用留白分组。
- 负值不用大红底，只保留红色数字，避免页面“风险告警化”。

### 3.7 区块 G：赛季尾注（Footer）

1. 升级目标
- 从一句鼓励文案，升级成“可收藏签名区”。

2. 内容
- `SEASON 档案编号`
- `本季结论一句话（自动生成）`
- `数据源与最后同步时间`
- `导出为海报 / 打印 A6` 操作入口（收藏闭环）

## 4. 动效策略（系统级）

1. 动效分层
- 功能动效：计数、展开、图表交互（必须）
- 气氛动效：撒花、闪烁、章纹（可选）
- 规则：同屏同时运行不超过 2 组气氛动效

2. 时序策略
- 0-300ms：页面骨架与 Hero 信息就位
- 300-1200ms：数值与时间轴渐进
- 1200ms 后：非关键装饰才出现

3. 完成态奖励
- 将一次性撒花改为“奖章解锁动画 + 轻粒子 1.2s”，更高级、更克制。
- 未完成态也给“阶段奖章”，避免强烈落差。

4. 无障碍与性能
- `prefers-reduced-motion` 下禁用粒子与弹跳，仅保留透明度过渡。
- 图表动画降采样，避免低端机掉帧；重绘放在 `requestAnimationFrame` 批次。

---

## 5. 信息架构与阅读路径

### 5.1 单屏优先级（从上到下）
1. 我现在成绩如何（进度 + 净值）
2. 我离目标还有多远（差额 + 预计次数/时间）
3. 我是怎么到这里的（趋势 + 每次战报）
4. 钱都花在哪了（结构）
5. 我能不能复核（明细账本）

### 5.2 文案语气
- 保留“战绩感”，减少“喊口号”。
- 采用“结论先行 + 证据补充”的句式，例如：
  - `本季已完成 78.4%，还差 ¥364。`
  - `近三次单次净值连续上升，回本节奏稳定。`

---

## 6. 落地建议（非代码）

1. 先做低风险改进（1 周）
- 合并 season + mini 模块
- 精简趋势图标注
- 统一色彩与徽章语义

2. 再做结构升级（2-3 周）
- 战报与账本改抽屉式层级
- 资金流向改版
- 增加导出/打印收藏路径

3. 设计验收指标
- 首屏 3 秒内可回答“是否回本、差多少”
- 用户完成一次“展开复盘”的比例提升
- 导出收藏按钮点击率（衡量“收藏级”是否成立）

---

## 7. 一句话结论

当前版本已经是“高完成度战绩票”；2.0 建议走 **Archive Pass（档案馆票证）**：保留票根灵魂，降低视觉噪声，强化长期收藏价值与复盘效率，让它从“好看的一次性截图”进化为“每个赛季都想保存的数字纪念物”。
