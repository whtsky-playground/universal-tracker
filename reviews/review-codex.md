# 设计评审（Codex）

基线版本（`src/index.html`）已经是高完成度作品：主题统一、数据链路完整、交互动效可用。三份方案的问题不是“没想法”，而是都有不同程度的“概念过满”。下面按“是否真能在单 HTML 做漂亮”来评。

## 1) 三份方案犀利点评

## Claude 方案（年卡护照）
优点：
- 方向最对：从“一次性战绩截图”升级为“长期收集叙事”（成就墙、护照页）是当前版本真正缺口。
- 能抓住关键痛点：回本后叙事断层、列表重复疲劳、分享场景单一。
- 对现有资产态度正确：保留 hero 灵魂，不推倒重来。

问题（不客气版）：
- 概念过度包装：护照、签证页、MRZ、真实条码、蜡封、烫金、路线图、翻页，全上会变主题乐园 PPT，不是产品页面。
- 交互野心偏大：叠卡翻页 + 成就动画 + SVG 路线逐点联动，在单文件里能做，但极易写成“动画很多、信息很慢”。
- 内容建模不够现实：如“购物达人>=3次”“连续4周”这类成就，当前数据结构未必稳定支持，容易出现规则写死和误判。

结论：方向一流，收敛不够。需要砍特效，保留叙事骨架。

## Codex 方案（Archive Pass）
优点：
- 最工程化：强调降噪、信息优先、色彩收敛、可访问性和性能预算，符合单 HTML 长期维护。
- 结构判断成熟：把 season + mini 合并、图表减标注、战报分层，都是“立刻见效”的改动。
- 对“收藏感”的理解更克制：不靠堆装饰，靠材质和排版秩序。

问题：
- 太安全：整体像“优秀改版规范”，但缺一个记忆点级创新（用户截图时不一定比现版更想发）。
- 情绪张力不足：强调克制后，可能把“游乐场快乐感”一起削掉，落成金融看板风。
- 有些提法仍偏重：例如“当日抽屉”“双模式图表切换”，在单文件里会加复杂度和状态管理成本。

结论：最可落地，但需要补一针“可传播的情绪设计”。

## Gemini 方案（荣耀勋章）
优点：
- 传播导向明确：知道用户想炫耀，hero 社交属性拉满。
- 仪式感思路有价值：达成时刻的奖励机制比纯数据更能激励复访。

问题（核心）：
- 方向跑偏：从“票根收藏”跳到“金属勋章/RPG 奖杯”，世界观断裂，和当前代码资产不连续。
- 大量提案是“想法很美实现很丑”：3D 浮雕字、金属拉丝、飞入 bounce、全屏爆闪，单 HTML 很容易廉价感爆炸。
- 可用性风险大：竖版徽章 + 环形主进度 + 横滑战报，会牺牲当前最强能力（纵向扫读效率）。

结论：适合拿来做“局部庆祝动效灵感”，不适合做主视觉架构。

## 2) 单 HTML 里哪些能做得好，哪些会翻车

## 真正能做好的（建议保留）
- Hero 保留票根结构，只做状态逻辑升级：100% 后主数值切“净赚金额”。
- mini-tickets 合并成一张“信息页”卡（三大数值 + 三个次级指标）。
- 成就系统做轻量网格（6~9 个），规则先用现有数据可直接计算的指标。
- 趋势图可继续 Canvas，但先做“减法”（少标签、固定 tooltip 区）再考虑 SVG。
- 入园战报保留纵向卡片，仅增加按月份折叠与 BEST/WORST 视觉区分。
- 分享增强做“预设截图区域”而不是引入 html2canvas 全链路。

## 能做但很容易“实现很丑”的
- 蜡封、烫金、金属、浮雕并用：材质一多就廉价。
- 护照翻页/叠卡强交互：移动端细节处理不好会卡顿且误触。
- 环形罗盘/复杂半环图：信息密度高时可读性会输给朴素条形图。
- 大量入场动画和粒子：短期惊艳，长期疲劳且掉帧。

## 不建议在当前约束下推进的
- “真实可扫码条码+公开跳转”作为核心能力（依赖部署、分享链路和容错，不是 UI 改版主线）。
- 重度 3D 视觉模拟（CSS 阴影堆叠式“伪高级”）。

## 3) 最佳融合方案（我建议的最终稿）

目标：**保留现版票根灵魂 + 引入可持续叙事 + 控制单文件复杂度**。

## 方案骨架（按区块）
1. Hero（沿用现版）
- 保留：票根、巨型数字、印章。
- 升级：
  - `<100%` 显示回本百分比。
  - `>=100%` 显示 `+¥surplus`，印章改为 `PROFIT` 分级。

2. 概览区（合并 season + mini）
- 上半：一条简洁里程碑进度（25/50/75/100）。
- 下半：三列关键金额 + 次级指标一行。
- 结果：减少一整个卡片区块，缓解长页面疲劳。

3. 成就墙（新增，轻量）
- 2x3 或 3x3 网格，不做复杂翻页。
- 只上“当前数据必能算”的成就：首次入园、5次入园、回本、单日最佳、总消费阈值、连续活跃（可选）。
- 已达成显示日期，未达成灰态。

4. 趋势图（先做减法）
- 保留 Canvas 代码框架，减少常驻标注。
- Tooltip 固定在图下信息条，避免遮挡曲线。
- 仅保留“回本点”一个强强调元素。

5. 战报与账本
- 战报保持竖向列表 + 月份折叠。
- 账本保留现有展开逻辑，增加“回本当天”锚点提示。

6. 分享
- 只做两个截图友好区：Hero、Hero+成就摘要。
- 不引入 html2canvas，先靠布局和背景控制提升原生截图质量。

## 视觉与动效原则
- 视觉：以当前纸质体系为主，只新增 1 个强调色（护照蓝或黄铜金二选一）。
- 动效：同屏最多 2 个并行动画；去掉大部分“飞入/爆炸”，保留计数和里程碑点亮。
- 性能：继续 `prefers-reduced-motion`，动画统一用 `transform/opacity`。

## 推荐实施顺序
- Phase 1（立即见效）：Hero 状态切换 + 概览合并 + 图表减标注。
- Phase 2（差异化）：轻量成就墙 + 战报分组折叠。
- Phase 3（锦上添花）：截图友好模板 + 少量材质细节升级。

一句话结论：**用 Claude 的叙事方向、Codex 的工程收敛、Gemini 的“达成时刻情绪点”做融合；砍掉重材质和重交互，才能在单 HTML 里做出“真高级”，而不是“高概念低质感”。**
